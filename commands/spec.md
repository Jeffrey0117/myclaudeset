規格驅動開發 (Spec-Driven Development)

目標功能：$ARGUMENTS

## 執行步驟

### 1. 建立規格文檔
- 在專案根目錄建立 `specs/` 資料夾（如不存在）
- 建立 `specs/<功能名稱>.md` 規格文檔
- 文檔結構：
  ```
  # <功能名稱>

  ## 概述
  簡述功能目的

  ## 需求
  - [ ] 需求 1
  - [ ] 需求 2
  ...

  ## 技術規格
  - 相關檔案
  - API 設計
  - 資料結構

  ## 驗收條件
  - [ ] 條件 1
  - [ ] 條件 2
  ```

### 2. 建立 Todo 追蹤
使用 TodoWrite 工具將規格中的需求和驗收條件轉為 todo 項目，方便追蹤進度。

### 3. 開發流程
- 每完成一個需求，更新規格文檔中的 checkbox
- 同步更新 TodoWrite 的狀態
- 確保代碼與規格一致

### 4. 完成確認
- 所有 checkbox 都打勾
- 所有 todo 都標記完成
- 規格文檔更新為最終狀態

IMPORTANT: 先寫規格，再寫代碼。規格是唯一真相來源 (Single Source of Truth)。
